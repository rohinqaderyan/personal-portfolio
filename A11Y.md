# Accessibility Guidelines

## Overview

This portfolio follows WCAG 2.1 Level AA standards to ensure accessibility for all users.

## Key Principles

### 1. Perceivable

**Color Contrast**

- Minimum 4.5:1 for normal text
- Minimum 3:1 for large text (18px+)
- Use tools: WebAIM Contrast Checker

**Alternative Text**

```tsx
<img src="/photo.jpg" alt="Rohin Qaderyan presenting at tech conference" />
<img src="/decorative.svg" alt="" role="presentation" />
```

**Semantic HTML**

```tsx
<nav>
  <ul>
    <li><a href="/">Home</a></li>
  </ul>
</nav>

<main>
  <article>
    <h1>Title</h1>
    <p>Content</p>
  </article>
</main>
```

### 2. Operable

**Keyboard Navigation**

- All interactive elements reachable via Tab
- Visible focus indicators
- Skip to main content link

**Focus Management**

```tsx
<button className="focus:outline-none focus:ring-2 focus:ring-primary" aria-label="Download resume">
  Download
</button>
```

**Touch Targets**

- Minimum 44x44 pixels
- Adequate spacing between interactive elements

### 3. Understandable

**Clear Labels**

```tsx
<label htmlFor="email">Email Address</label>
<input
  id="email"
  type="email"
  aria-required="true"
  aria-describedby="email-help"
/>
<span id="email-help">We'll never share your email</span>
```

**Error Messages**

```tsx
;<input aria-invalid={hasError} aria-describedby={hasError ? 'email-error' : undefined} />
{
  hasError && (
    <span id="email-error" role="alert">
      Please enter a valid email address
    </span>
  )
}
```

### 4. Robust

**ARIA Landmarks**

```tsx
<header role="banner">
  <nav role="navigation" aria-label="Main navigation">
  </nav>
</header>

<main role="main">
  <section aria-labelledby="projects-heading">
    <h2 id="projects-heading">Projects</h2>
  </section>
</main>

<footer role="contentinfo">
</footer>
```

## Testing

### Automated Testing

```bash
# Run accessibility tests
npm run test:a11y

# Lighthouse accessibility audit
npm run lighthouse
```

### Manual Testing

1. **Keyboard Only**
   - Navigate using only Tab, Enter, Space
   - Check all features are accessible

2. **Screen Reader**
   - NVDA (Windows) - Free
   - JAWS (Windows) - Paid
   - VoiceOver (Mac) - Built-in

3. **Browser Extensions**
   - axe DevTools
   - WAVE
   - Lighthouse

### Checklist

- [ ] All images have alt text
- [ ] Color contrast meets WCAG AA
- [ ] Keyboard navigation works
- [ ] Focus visible on all interactive elements
- [ ] Forms have labels and error messages
- [ ] Headings follow logical structure (h1 → h2 → h3)
- [ ] ARIA landmarks used correctly
- [ ] No flashing content (seizure risk)
- [ ] Text can be resized to 200%
- [ ] Screen reader friendly

## Common Issues & Fixes

### Missing Alt Text

```tsx
// ❌ Bad
<img src="/project.jpg" />

// ✅ Good
<img src="/project.jpg" alt="AI Analytics Dashboard showing real-time metrics" />
```

### Poor Color Contrast

```tsx
// ❌ Bad - Light gray on white
<span className="text-gray-300">Important text</span>

// ✅ Good - Dark gray on white
<span className="text-gray-700">Important text</span>
```

### Missing Labels

```tsx
// ❌ Bad
<input type="text" placeholder="Email" />

// ✅ Good
<label htmlFor="email">Email</label>
<input id="email" type="text" placeholder="you@example.com" />
```

### Non-Semantic HTML

```tsx
// ❌ Bad
<div onClick={handleClick}>Click me</div>

// ✅ Good
<button onClick={handleClick}>Click me</button>
```
