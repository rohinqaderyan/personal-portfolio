## Reporting a Vulnerability

If you discover a security vulnerability, please email ahmad.qaderyan@pfizer.com with:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

**Please do not open public issues for security vulnerabilities.**

## Security Best Practices

This portfolio follows these security practices:

### Content Security Policy (CSP)

- Strict CSP headers configured
- SVG sanitization enabled
- XSS protection headers

### Dependencies

- Regular updates via Dependabot
- Automated security audits
- No known vulnerabilities

### Data Protection

- No sensitive data stored
- HTTPS-only in production
- Secure headers configured

### Authentication

- No authentication required
- No user data collected
- Privacy-friendly analytics only

## Updates

Security updates are released as soon as possible after discovery.
